What Happens During Encryption?
The question paper goes through a 4-step cryptographic pipeline entirely in your browser before anything is stored:
1
AES-256-GCM symmetric encryption
A random 256-bit key encrypts the paper. The output is unreadable binary — no one can see the questions.
2
RSA-2048-OAEP key wrapping
The AES key is itself encrypted using RSA. Only the holder of the RSA private key (stored in the exam record) can unwrap it.
3
SHA-256 integrity hash
A fingerprint of the original plaintext is stored. If anyone modifies the ciphertext, this hash will not match on decryption.
4
Time-locked access control
Even though the sealed package is in sessionStorage, decryption is blocked until the exam window opens. No key = no content.

Is the paper transferred?

              In this demo: no network transfer — everything stays in your browser's sessionStorage. In a real system, only the sealed ciphertext package would be sent to a server. Students download the sealed package, but it stays locked until the exam time.

              
Why encrypt if students can't open it anyway?

              Access control can be bypassed — a rogue admin, SQL injection, or a server breach could expose the file. Encryption means even a raw database dump reveals nothing readable. The questions are only decryptable with the private key at exam time.
